### How to Use:

**命令行工具**:
```bash
# 查看统计
./venv/bin/python tools/message_query.py stats

# 查询最近50条消息
./venv/bin/python tools/message_query.py query --limit 50

# 导出最近1小时消息为JSON
./venv/bin/python tools/message_query.py export --hours 1 --format json --output messages.json

# 清理30天前的消息
./venv/bin/python tools/message_query.py clean --days 30

# 归档(而非删除)30天前的消息
./venv/bin/python tools/message_query.py clean --days 30 --archive

# 优化数据库
./venv/bin/python tools/message_query.py optimize
```

**REST API**:
```bash
# 获取统计
curl http://localhost:8000/api/messages/stats

# 获取消息趋势
curl http://localhost:8000/api/messages/stats/trend?interval_minutes=60&limit=24

# 导出消息
curl -X POST "http://localhost:8000/api/messages/export?format=json&hours=1" \
  --output messages.json

# 清理旧消息
curl -X POST "http://localhost:8000/api/messages/clean?days=30&archive=false"

# 优化数据库
curl -X POST http://localhost:8000/api/messages/optimize
```

### Notes:
- 批量写入使用事务保证原子性
- 归档功能保留数据用于历史分析
- 导出文件自动清理,只保留最近10个
- 数据库优化建议定期运行(如每周一次)
- 所有API端点支持完整的CORS配置

### Default Alert Rules:
1. CPU > 80% (WARNING, 120s)
2. CPU > 95% (CRITICAL, 60s)
3. Memory > 85% (WARNING, 120s)
4. Memory > 95% (CRITICAL, 60s)
5. Disk > 90% (ERROR, 300s)
6. Unhealthy Agents > 0 (WARNING, 60s)
7. Throughput < 0.1 msg/s (INFO, 300s)
