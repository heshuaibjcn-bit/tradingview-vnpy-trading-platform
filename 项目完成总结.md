## 📖 使用示例

### 场景 1：技术指标自动交易

**TradingView 设置**：
```
指标: RSI
触发条件: RSI < 30 (超卖买入)
Webhook: http://your-ip:5000/webhook
消息: {"symbol":"600000","action":"buy","price":{{close}}}
```

**自动执行**：
1. TradingView 检测到 RSI < 30
2. Webhook 发送警报到服务器
3. vnpy 自动创建买入订单
4. 订单发送到东财接口执行

### 场景 2：趋势跟踪策略

**TradingView 设置**：
```
工具: 趋势线突破
触发条件: 价格突破趋势线
Webhook: http://your-ip:5000/webhook
消息: {"symbol":"000001","action":"buy","price":{{close}}}
```

**自动执行**：
1. 手动在图表画趋势线
2. 价格突破时触发警报
3. 系统自动开仓/平仓
4. 实时跟踪趋势变化

## 🔧 技术特性

### 前端技术
- **PyQt6** - 桌面 GUI 框架
- **QWebEngineView** - 浏览器组件
- **TradingView Widget** - 图表组件

### 后端技术
- **vnpy 4.0** - 量化交易框架
- **Flask** - Web 框架
- **requests** - HTTP 库

### 数据流
```
TradingView → HTTP Webhook → Flask 服务器
    ↓
vnpy MainEngine → 事件引擎 → 订单管理
    ↓
东财网关 → HTTP API → 交易所
```

## 🐛 已知问题

### 1. TradingView 部分 API 需要付费
- **影响**: 免费版 Webhook 功能有限
- **解决**: 使用邮件转 Webhook 方案

### 2. 东财 SDK Python 版本限制
- **影响**: 官方 SDK 只支持 Python 2.7-3.8
- **解决**: 使用 HTTP API 方案，支持所有版本

### 3. 部分 API 端点可能失效
- **影响**: 行情数据查询可能失败
- **解决**: 定期更新 API 地址
